You are a senior full-stack engineer building a production-ready SaaS application.

Project Name: CarbonFlow (Working Name)

## 🎯 PRODUCT PURPOSE

Build a Phase 1 (Version 1) system for a Carbon Accounting / Supply Chain Data Platform.

The goal is NOT to build a full ESG platform yet.

The goal is to solve ONE core problem:

👉 Enable companies to collect carbon-related data from their suppliers (Scope 3) and generate a basic emissions report.

This system must be clean, scalable, and structured for future expansion.

---

## 🧠 CORE CONCEPT

There are two types of users:

1. Company (Main User)
2. Supplier (External, no login required)

Flow:

* Company signs up
* Company adds suppliers
* Suppliers receive a link
* Suppliers submit data via a simple form
* System calculates emissions
* Company sees results in dashboard
* Company downloads a report

---

## 🛠️ TECH STACK (MANDATORY)

Frontend:

* React (functional components)
* Tailwind CSS

Backend:

* Node.js with Express.js

Authentication:

* Firebase Authentication (Email/Password login)

Database:

* PostgreSQL using Neon (serverless)

Other:

* REST API architecture
* Clean folder structure (controllers, services, routes, models)
* jsPDF for report generation

---

## 🧱 SYSTEM ARCHITECTURE

Follow modular architecture:

/controllers → handle requests
/services → business logic (emissions calculation)
/routes → API routes
/models → database queries
/utils → helpers

---

## 🗄️ DATABASE DESIGN

Tables:

1. companies

* id (UUID)
* name
* email
* created_at

2. users

* id
* company_id
* email
* role

3. suppliers

* id
* company_id
* name
* email_or_phone
* status (pending/completed)

4. emissions

* id
* supplier_id
* electricity_usage
* fuel_usage
* transport_distance
* total_co2
* created_at
